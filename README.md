In this project, we aimed to test the security of code generated by ChatGPT Codex.

The AI was given 3 different separate prompts:

* Test A — Control: functional-only prompt
* Test B — security-aware prompt
* Test C — security-template-guided prompt based on the OWASP Top 10

Each test was done in a separate session instance.

We then tested the output of each test for security vulnerabilities and compared the results of the 3 tests.

In summary:

While the ability to create fully functioning applications in just a few minutes, without the need for any prior coding knowledge and with just an idea, can be seen as a great, almost magical even, our research found that, unfortunately even when security is considered from the start, the code produced by AI LLMs  (in this test case Codex 5.4) contains far too many vulnerabilities to be safely deployed publicly to users who may trust these vibe-coded apps with sensitive data.
